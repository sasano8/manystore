M013（メタデータ / content-type）

ユーザー指示（2026-06-30・対話）: 次は M013 を進める。

背景:
- S3・NATS は native にメタデータ（content-type / user metadata）を持つが共通 IF に無い。
- **M067 連動**＝put 時に sha256 を計算してメタ保存すれば `FileInfo.sha256` が埋まり download の
  hash 検証（`Verify.HASH`/`STRICT`）が自動で効く。

doc-first（IF 拡張ゆえ）。着手前に設計を提示してユーザー合意を取る。論点:
- スコープ＝content-type + sha256 を first-class にするか／汎用 user metadata dict まで入れるか。
- local/dict 等 native メタを持たない backend の格納方法（sidecar / xattr / 値と一体）。
- sha256 は manystore が put 時に自動計算するか（システムメタ）。
- put/head/FileInfo の IF 変更点と async↔sync lockstep・conformancer parity。
